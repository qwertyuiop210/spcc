The Tower of Hanoi is a problem consisting of three rods (A, B, C) and n disks of different sizes.
The objective is to move all disks from the source rod to the destination rod following rules:

Only one disk can be moved at a time
A larger disk cannot be placed on a smaller disk
Only the top disk of a rod can be moved
🔹 Hill Climbing Algorithm

Hill Climbing is a local search algorithm that:

Evaluates neighboring states
Selects the state with the best heuristic value
Moves step-by-step toward the goal

It does not consider the full path, only immediate improvement.

🔹 State Representation
Each state is represented as:
[Rod1, Rod2, Rod3]

Example:

[[3,2,1], [], []]
🔹 Heuristic Function
h(n)=Number of disks in goal rod
Goal rod = third rod
Higher value = closer to goal
🔹 Working
Start with all disks on source rod
Generate all possible valid moves
Evaluate each state using heuristic
Move to the state with highest heuristic value
Repeat until:
Goal is reached OR
No better state exists
🔹 Advantages
Simple and easy to implement
Requires less memory
Faster for small problems
🔹 Disadvantages
May get stuck in local optimum
Cannot backtrack
Not guaranteed to find solution
Not suitable for problems needing global planning
🔹 Limitation in Tower of Hanoi

Hill Climbing often fails because:

Some moves temporarily reduce heuristic value
Algorithm does not allow such moves
Hence, it gets stuck before reaching goal


Hill Climbing solves Tower of Hanoi by selecting states with better heuristic values but may fail due to local optimum.
